# プロジェクト管理・業務負荷管理システム

プロジェクトの進行管理、タスク管理、業務負荷管理、ルーティンジョブの工数管理を統合したWebアプリケーションです。

## 🌟 主要機能

### 📊 ダッシュボード
- プロジェクト・タスク・工数・メンバーのKPI表示
- プロジェクト状況とタスク進捗のチャート表示
- 最近のアクティビティ一覧

### 📈 ガントチャート
- プロジェクト・タスクの時系列表示
- 担当者・プロジェクト別のフィルタリング
- 直感的な進捗管理

### ⚖️ 業務負荷管理
- **8時間警告機能**: 1日の業務負荷が8時間を超える場合に警告表示
- **10時間破綻メッセージ**: 1日の業務負荷が10時間を超える場合に破綻警告
- デイリーの業務負荷トレンド表示
- メンバー別負荷状況の詳細テーブル
- タスク作成時の自動負荷チェック

### 📅 カレンダー機能
- プロジェクトとタスクのカレンダー表示
- マイルストーンの可視化
- 月次・週次・リスト表示対応

### 📋 タスク管理
- タスクの作成・編集・削除機能
- プロジェクト・担当者・ステータス・優先度によるフィルタリング
- 依存関係の管理

### 🔄 ルーティンジョブ管理
- 定期的な業務の工数管理
- 曜日別実行設定
- 担当者別工数統計

### ⚙️ システム管理
- プロジェクトとメンバーの管理
- 色分けによる視覚的識別
- データの一元管理

## 🛠 技術スタック

## 🎯 業務負荷管理の特徴

### 自動負荷計算
システムは以下の要素を自動的に計算して業務負荷を算出します：

1. **プロジェクトタスク工数**
   - タスクの予定工数を期間で割った1日あたりの工数

2. **ルーティンジョブ工数**
   - 各曜日に設定されたルーティン業務の工数

### 負荷警告システム
- **8時間以上**: ⚠️ 警告レベル（業務負荷が高くなっています）
- **10時間以上**: 🚨 破綻レベル（業務負荷が限界を超えています）

### タスク作成時の事前チェック
新規タスク作成時に、期間中の各日の業務負荷を事前計算し、警告レベルに達する場合は確認ダイアログを表示します。

## 📊 API エンドポイント

### データ取得
- `GET /api/data` - 全データ取得
- `GET /api/members` - メンバー一覧
- `GET /api/projects` - プロジェクト一覧
- `GET /api/tasks` - タスク一覧
- `GET /api/routine-jobs` - ルーティンジョブ一覧

### 業務負荷関連
- `GET /api/workload/:memberId` - 特定メンバーの業務負荷
- `GET /api/workload-all` - 全メンバーの業務負荷

### ガントチャート・カレンダー
- `GET /api/gantt` - ガントチャート用データ
- `GET /api/calendar` - カレンダー用イベント

### データ操作
- `POST/PUT/DELETE /api/projects/:id` - プロジェクト管理
- `POST/PUT/DELETE /api/tasks/:id` - タスク管理
- `POST/PUT/DELETE /api/members/:id` - メンバー管理
- `POST/PUT/DELETE /api/routine-jobs/:id` - ルーティンジョブ管理

## 🚀 使用方法

### 基本的なワークフロー

1. **メンバー登録**: システム管理でチームメンバーを登録
2. **プロジェクト作成**: プロジェクトの基本情報を設定
3. **タスク作成**: プロジェクトに関連するタスクを作成（業務負荷自動チェック）
4. **ルーティンジョブ設定**: 定期的な業務の工数を設定
5. **進捗管理**: ガントチャートとカレンダーで進捗を監視
6. **負荷管理**: 業務負荷タブで各メンバーの負荷状況を確認

### 業務負荷管理の活用

1. **タスク作成時**: 自動的に業務負荷をチェックし、過負荷の場合は警告
2. **日次確認**: 業務負荷トレンドで各メンバーの負荷状況を監視
3. **調整**: 過負荷が予想される場合は、タスクの再配分や期間調整を実施
